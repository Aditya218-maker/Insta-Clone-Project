Tumhara feature: Ek user dusre user ko follow kar sakta hai

Example:

A follows B
A follows C
D follows A

Ab hame store karna hai: A kisko follow karta hai aur A ko kaun follow karta hai
Ek user multiple users ko follow kar sakta hai

To kyunki: ✅ ek user → multiple users

isliye structure hona chahiye:  one-to-many relationship

iska pehla aproach hai user.model.js me jaake do arrya create krle followers aur Following

MongoDB me multiple values rakhne ke liye: ✅ array use karte hain

Followers : [{
   type: mongoose.Schema.Types.ObjectId,
   ref: "users"
}],
Following : [{
   type: mongoose.Schema.Types.ObjectId,
   ref: "users"
}]

ObjectId kyun? Hum poora user object store nahi karte.

ref: "users" : Taaki mongoose ko pata chale: Ye ids users collection ki hain
